## 1. Введение

**KapyHolograms** — современный плагин голограмм для серверов на базе Paper. Для отображения используются нативные сущности `TEXT_DISPLAY` (появились в Minecraft 1.20.5) — никакого Armor Stand под капотом, только чистое, актуальное рендеринг-API.

### Основные возможности

| Категория | Описание |
|---|---|
| 🖥️ Рендеринг | Нативные сущности `TEXT_DISPLAY` — текст, предметы, головы |
| 👁️ FOV-отсечение | Голограммы отправляются только тем игрокам, которые смотрят на них |
| 📍 Пространственный индекс | Поиск O(1) по чанкам — работает с тысячами голограмм |
| ⚡ Событийная видимость | Показ/скрытие запускается по событию движения, а не глобальным тиком |
| 📏 LOD-обновления | Далёкие голограммы обновляются реже, что экономит процессор |
| 📄 Хранение по файлам | Каждая голограмма — отдельный `.yml`-файл в папке `holograms/` |
| 🖱️ Кликабельные строки | 6 типов действий: команда, консольная команда, сообщение, URL, телепорт, переключение страницы |
| 🎭 Мультистраничность | Несколько страниц на голограмму с автоматическим или ручным переключением |
| 🎨 Богатое форматирование | MiniMessage (градиенты, HEX-цвета) **или** Legacy (`&`-коды) — на выбор |
| 🔧 PlaceholderAPI | Поддержка плейсхолдеров для каждого игрока с LOD-зависимым обновлением |
| 🌐 Локализация | Встроенные переводы: английский, русский, украинский |
| 📦 Чистый API | Fluent-API с построителем для разработчиков плагинов |

## 2. Требования и установка

### Требования

- **Сервер:** Paper 1.20.5 или новее (совместимость с Folia не проверялась)
- **Java:** 21 или новее
- **Опционально:** [PlaceholderAPI](https://www.spigotmc.org/resources/placeholderapi.6245/) для поддержки `%плейсхолдеров%` в тексте голограмм

### Установка

1. Скачайте `KapyHolograms.jar` со страницы [Releases](../../releases).
2. Поместите файл в папку `plugins/` вашего сервера.
3. Запустите (или перезапустите) сервер — плагин создаст файлы конфигурации автоматически.
4. Отредактируйте `plugins/KapyHolograms/config.yml` по необходимости, затем выполните `/kh reload`.

### Обновление

Просто замените старый `.jar` новым и перезапустите сервер. Файлы данных голограмм из папки `holograms/` сохраняются между версиями.

## 3. Справочник команд

Все команды используют корневую команду `/kapyholograms` (псевдоним: `/kh`).

> **Совет:** Автодополнение по Tab поддерживается для имён голограмм и числовых индексов.

### 3.1 Управление голограммами

| Команда | Описание | Право |
|---|---|---|
| `/kh create <имя>` | Создать голограмму на текущей позиции | `kapyholograms.command.create` |
| `/kh delete <имя>` | Безвозвратно удалить голограмму и её файл | `kapyholograms.command.delete` |
| `/kh list [страница]` | Показать список всех голограмм (по 10 на странице) | `kapyholograms.command.list` |
| `/kh info <имя>` | Показать информацию: координаты, страницы, строки, дистанция | `kapyholograms.command.info` |
| `/kh teleport <имя>` | Телепортироваться к голограмме | `kapyholograms.command.teleport` |
| `/kh movehere <имя>` | Переместить голограмму к вашей текущей позиции | `kapyholograms.command.movehere` |
| `/kh nearest` | Найти и показать ближайшую голограмму | `kapyholograms.command.nearest` |
| `/kh edit <имя>` | Открыть интерактивное GUI-меню редактирования | `kapyholograms.command.edit` |

### 3.2 Управление строками

Строки нумеруются с 1. Первая добавленная строка имеет индекс `1`.

| Команда | Описание |
|---|---|
| `/kh line add <голо> <текст>` | Добавить строку в конец страницы 1 голограммы |
| `/kh line insert <голо> <индекс> <текст>` | Вставить строку **перед** указанным индексом |
| `/kh line set <голо> <индекс> <текст>` | Заменить строку с указанным индексом |
| `/kh line remove <голо> <индекс>` | Удалить строку с указанным индексом |

**Все команды строк требуют** `kapyholograms.command.line`.

**Примеры:**
```
/kh line add welcome <gradient:#ff5555:#ffaa00>Добро пожаловать!</gradient>
/kh line add welcome <gray>Игроков онлайн: <white>%server_online%
/kh line insert welcome 1 <gold>>>> Добро пожаловать на сервер! <<<
/kh line set welcome 2 <gray>Онлайн: <aqua>%server_online%
/kh line remove welcome 3
```

### 3.3 Управление страницами

Страницы нумеруются с 1. Каждая голограмма создаётся с одной страницей.

| Команда | Описание |
|---|---|
| `/kh page add <голо>` | Добавить пустую страницу к голограмме |
| `/kh page remove <голо> <индекс>` | Удалить страницу с указанным индексом (последнюю страницу удалить нельзя) |
| `/kh page switch <голо> <индекс>` | Переключить голограмму на указанную страницу для всех зрителей |

**Все команды страниц требуют** `kapyholograms.command.page`.

### 3.4 Утилиты

| Команда | Описание | Право |
|---|---|---|
| `/kh reload` | Перезагрузить `config.yml`, все файлы сообщений и данных голограмм | `kapyholograms.reload` |
| `/kh help [страница]` | Показать постраничное меню помощи | *(не требуется)* |

## 5. config.yml — Полный справочник

Главный файл конфигурации находится по пути `plugins/KapyHolograms/config.yml`.

```yaml
# ─────────────────────────────────────────────
#  KapyHolograms — Основная конфигурация
# ─────────────────────────────────────────────

# Язык сообщений плагина.
# Доступные значения: en (английский), ru (русский), uk (украинский)
language: ru

# Формат текста для СОДЕРЖИМОГО СТРОК голограмм.
# Не влияет на сообщения плагина (они всегда используют MiniMessage).
#
# minimessage — современный формат Adventure:
#   <gold>Текст</gold>  <gradient:#ff0000:#00ff00>Текст</gradient>  <#ff5500>Текст</#ff5500>
#
# legacy — классический формат Bukkit/Essentials:
#   &6Золото  &lЖирный  &#ff5500Hex  &r Сброс
text-format: minimessage

# ─── Производительность ───────────────────────
performance:

  # Отправлять голограммы только игрокам, которые действительно смотрят на них.
  # Значительно снижает количество пакетов в зонах с множеством голограмм.
  fov-culling: true

  # Половинный угол конуса поля зрения для отсечения (градусы).
  # 60  = строго (только при прямом взгляде на голограмму)
  # 90  = стандарт (в пределах 180° фронтальной дуги)
  # 120 = мягко (даже при небольшом отведении взгляда)
  fov-angle: 120.0

  # Дистанция отображения голограмм по умолчанию (блоки).
  # Отдельные голограммы могут переопределять это значение через GUI или файл.
  default-render-distance: 48.0

  # Level-of-Detail (LOD) — обновление голограмм с разной частотой в зависимости от расстояния.
  lod:
    enabled: true

    # Голограммы ближе near-distance обновляются каждые near-update-ticks тиков.
    near-distance: 10.0
    # Голограммы ближе mid-distance (но дальше near) обновляются каждые mid-update-ticks тиков.
    mid-distance: 30.0
    # Голограммы дальше mid-distance обновляются каждые far-update-ticks тиков.

    near-update-ticks: 2    # ~0.1 с при 20 TPS
    mid-update-ticks: 20    # ~1 с  при 20 TPS
    far-update-ticks: 60    # ~3 с  при 20 TPS

  # Количество тиков между последовательными пересчётами видимости для одного игрока.
  # Увеличьте, чтобы снизить нагрузку на процессор; уменьшите для более отзывчивого показа/скрытия.
  visibility-throttle-ticks: 2

# ─── Настройки голограмм по умолчанию ────────
hologram-defaults:

  # Режим billboard (поворота) для новых голограмм.
  # CENTER     — всегда смотрит на игрока (стандарт)
  # HORIZONTAL — остаётся вертикальным, вращается только горизонтально
  # VERTICAL   — может наклоняться к игроку вертикально, но не горизонтально
  # FIXED      — не вращается; использует угол fixed-yaw
  billboard: CENTER

  # Добавлять ли тень к тексту голограммы.
  text-shadow: true

  # Прозрачность фона по умолчанию (0 = полностью прозрачный, 255 = полностью непрозрачный чёрный).
  background-color: 0

  # Вертикальный отступ между строками (в блоках).
  line-spacing: 0.3

# ─── Хранилище ────────────────────────────────
storage:
  # Подпапка внутри plugins/KapyHolograms/, где хранятся файлы голограмм.
  holograms-folder: holograms

# ─── PlaceholderAPI ───────────────────────────
placeholders:
  # Установите false, чтобы отключить PlaceholderAPI даже при его наличии.
  enabled: true
```

### Быстрый справочник: важные настройки

| Параметр | Описание | По умолчанию |
|---|---|---|
| `language` | Язык интерфейса (`en`, `ru`, `uk`) | `en` |
| `text-format` | Парсер текста строк (`minimessage` или `legacy`) | `minimessage` |
| `performance.fov-culling` | FOV-отсечение голограмм | `true` |
| `performance.fov-angle` | Половинный угол FOV-конуса (градусы) | `120.0` |
| `performance.default-render-distance` | Дальность видимости голограмм (блоки) | `48.0` |
| `performance.lod.enabled` | LOD-масштабирование частоты обновлений | `true` |
| `hologram-defaults.line-spacing` | Расстояние между строками (блоки) | `0.3` |
| `hologram-defaults.text-shadow` | Тень текста | `true` |
| `hologram-defaults.background-color` | Прозрачность фона (0–255) | `0` |

## 6. Форматирование текста: MiniMessage и Legacy

KapyHolograms поддерживает **два формата текста** для содержимого строк голограмм. Выбор осуществляется через `text-format` в `config.yml`.

> **Примечание:** Сообщения плагина (команды, чат, подсказки GUI) всегда используют MiniMessage независимо от этой настройки.

### 6.1 MiniMessage (по умолчанию, рекомендуется)

MiniMessage — современный язык разметки библиотеки Adventure. Поддерживает градиенты, HEX-цвета, события наведения/клика и многое другое.

| Тег | Результат |
|---|---|
| `<red>текст</red>` | Красный текст |
| `<gold>текст` | Золотой текст (автозакрытие) |
| `<#ff5500>текст` | HEX-цвет (краткая запись) |
| `<color:#ff5500>текст</color>` | HEX-цвет (явная запись) |
| `<bold>текст</bold>` | Жирный |
| `<italic>`, `<underlined>`, `<strikethrough>`, `<obfuscated>` | Оформление |
| `<gradient:#ff0000:#0000ff>текст</gradient>` | Цветовой градиент |
| `<rainbow>текст</rainbow>` | Радужный градиент |
| `<reset>` | Сбросить всё форматирование |

**Примеры:**
```
<gradient:#ff5555:#ffaa00><bold>Добро пожаловать!</bold></gradient>
<gray>Игроков онлайн: <white>%server_online%
<#55ff55>✔ <white>VIP-Сервер
```

Полный справочник MiniMessage: https://docs.advntr.dev/minimessage/format.html

### 6.2 Legacy (`text-format: legacy`)

Legacy-формат использует `&` в качестве символа форматирования (совместим с Essentials, CMI и т.д.).

| Код | Результат |
|---|---|
| `&0` – `&9`, `&a` – `&f` | Стандартные 16 цветов Minecraft |
| `&l` | Жирный |
| `&o` | Курсив |
| `&n` | Подчёркнутый |
| `&m` | Зачёркнутый |
| `&k` | Обфусцированный |
| `&r` | Сброс |
| `&#RRGGBB` | HEX-цвет (расширение Adventure) |

**Примеры:**
```
&#ff5500&lДобро пожаловать!&r
&7Игроков онлайн: &f%server_online%
&a✔ &fVIP-Сервер
```

## 7. Типы строк

Каждая строка голограммы имеет тип, определяющий способ её отображения.

| Тип | Как создать | Описание |
|---|---|---|
| `TEXT` | `/kh line add <голо> <текст>` | Текстовая строка, разбираемая согласно `text-format` |
| `ITEM` | Указать `type: ITEM` в файле голограммы | Парящий предмет (например, `DIAMOND_SWORD`) |
| `HEAD` | Указать `type: HEAD` в файле голограммы | Парящая голова игрока или кастомная голова |
| `EMPTY` | Указать `type: EMPTY` в файле голограммы | Невидимый разделитель-отступ |

> В настоящее время строки типов `ITEM`, `HEAD` и `EMPTY` можно добавить только через прямое редактирование YAML-файла голограммы. Команда `/kh line add` всегда создаёт строки типа `TEXT`.

### Динамические строки

Текстовая строка считается **динамической** (обновляется для каждого игрока), если её содержимое содержит символ `%`. Динамические строки вычисляются отдельно для каждого зрителя с использованием PlaceholderAPI (если установлен). Статические строки вычисляются один раз и переиспользуются — это эффективнее.

## 8. Типы голограмм

**Тип голограммы** определяет ориентацию отображаемой сущности в мире.

| Тип | Описание | Лучшее применение |
|---|---|---|
| `VERTICAL` | Голограмма всегда повёрнута к игроку (billboard: `CENTER`) | Стандартный парящий текст |
| `FIXED` | Голограмма смотрит в фиксированном направлении, заданном `fixed-yaw` | Вывески, стены, направленные дисплеи |

### Справочник угла FIXED

| `fixed-yaw` | Направление, куда смотрит голограмма |
|---|---|
| `0` | На юг (+Z) |
| `90` | На запад (−X) |
| `180` | На север (−Z) |
| `270` | На восток (+X) |

### Двусторонняя голограмма (только FIXED)

При `double-sided: true` с противоположной стороны создаётся зеркальная копия голограммы — текст читается как спереди, так и сзади. Применимо только при `type: FIXED`.

## 9. Действия по клику

Строки могут иметь **действия по клику**, выполняемые при нажатии правой кнопкой мыши на голограмму. К одной строке можно привязать несколько действий — они выполняются по порядку.

> Для срабатывания кликов у игрока должно быть право `kapyholograms.use` (по умолчанию: `true` для всех).

### Типы действий

| Тип | Формат `value` | Описание |
|---|---|---|
| `RUN_COMMAND` | `say Привет` | Выполняет команду от имени кликнувшего игрока |
| `CONSOLE_COMMAND` | `op Steve` | Выполняет команду через консоль |
| `SEND_MESSAGE` | `<green>Привет!` | Отправляет игроку сообщение в формате MiniMessage |
| `OPEN_URL` | `https://example.com` | Отправляет игроку кликабельную ссылку |
| `TELEPORT` | `world,100,64,200,0,0` | Телепортирует игрока (`мир,x,y,z[,yaw,pitch]`) |
| `SWITCH_PAGE` | `2` | Переключает голограмму на указанную страницу для этого игрока |
| `NONE` | *(любое)* | Пустое действие; используется как заглушка |

### Добавление действий через YAML

Действия добавляются путём прямого редактирования файла голограммы (команды для этого пока нет):

```yaml
# plugins/KapyHolograms/holograms/shop.yml
pages:
  - lines:
      - type: TEXT
        content: "<gold><bold>Нажмите, чтобы открыть магазин!"
        actions:
          - type: RUN_COMMAND
            value: shop
          - type: SEND_MESSAGE
            value: "<green>Открываю магазин..."
```

## 10. Формат файла голограммы

Каждая голограмма хранится в отдельном YAML-файле в папке `plugins/KapyHolograms/holograms/`.

### Полный пример

```yaml
# plugins/KapyHolograms/holograms/welcome.yml

name: welcome
world: world
x: 0.5
y: 64.0
z: 0.5

# Переопределение дистанции отображения (-1 = брать из config.yml)
render-distance: -1.0

enabled: true

# VERTICAL или FIXED
type: VERTICAL

# CENTER, LEFT или RIGHT
text-alignment: CENTER

# Прозрачность фона (0 = прозрачный, 255 = непрозрачный)
background-alpha: 0

# Интервал обновления содержимого в секундах (0 = LOD из конфига)
update-interval: 0

# Интервал автопереключения страниц в секундах (0 = выключено)
page-interval: 5

# Используется только при type: FIXED. Направление по компасу в градусах.
# 0=Юг  90=Запад  180=Север  270=Восток
fixed-yaw: 0.0

# Зеркальная копия сзади (только type: FIXED)
double-sided: false

pages:
  - lines:
      - type: TEXT
        content: "<gradient:#ff5555:#ffaa00><bold>Добро пожаловать!</bold></gradient>"
      - type: TEXT
        content: "<gray>Игроков онлайн: <white>%server_online%"
        actions:
          - type: RUN_COMMAND
            value: list
      - type: EMPTY
      - type: ITEM
        content: DIAMOND
  - lines:
      - type: TEXT
        content: "<gold>Страница 2 — Подробнее"
      - type: TEXT
        content: "<gray>IP сервера: <white>play.example.com"
```

### Справочник полей

| Поле | Тип | Описание |
|---|---|---|

## 11. Интеграция с PlaceholderAPI

KapyHolograms интегрируется с [PlaceholderAPI](https://www.spigotmc.org/resources/placeholderapi.6245/) для отображения индивидуальных значений для каждого игрока в строках голограмм.

### Настройка

1. Установите PlaceholderAPI на сервер.
2. Установите нужные расширения PAPI (например, `Player`, `Server`, `Vault` и т.д.).
3. Убедитесь, что `placeholders.enabled: true` в `config.yml`.
4. Используйте синтаксис `%плейсхолдер%` в текстовых строках голограмм.

### Принцип работы

- Строка с символом `%` помечается как **динамическая**.
- Динамические строки вычисляются отдельно для каждого игрока, видящего голограмму.
- LOD-система управляет тем, насколько часто динамические строки обновляются в зависимости от расстояния.
- Статические строки вычисляются один раз и используются общими для всех игроков — это эффективнее.

### Примеры

```
<gray>Добро пожаловать, <gold>%player_name%<gray>!
<gray>Ваш пинг: <white>%player_ping% мс
<gray>Онлайн: <white>%server_online%/<gray>%server_max%
<gray>Ваш баланс: <gold>$%vault_eco_balance%
```

## 12. GUI-меню редактирования

GUI-меню редактирования предоставляет удобный интерфейс для настройки всех параметров голограммы прямо в игре — без редактирования YAML-файлов.

### Открытие меню

```
/kh edit <имя>
```

Требует право `kapyholograms.command.edit` (по умолчанию: `op`).

### Схема меню

```
┌───────────────────────────────────────────────────────┐
│  [Пан] [Пан] [Пан] [Пан] [Пан] [Пан] [Пан]...        │  ← Ряд 0 (заголовок)
│  [Пан] [Вкл] [Тип] [Вырн] [Пан] [Фон] [Пан]          │  ← Ряд 1 (основные настройки)
│        [Сюда]                                          │
│  [Пан] [Обн] [Стр] [Пан] [Дист] [Пан] [Пан]...       │  ← Ряд 2 (интервалы)
│  [Пан] [Угол] [2Ст] [Пан] ....                        │  ← Ряд 3 (настройки FIXED)
│  [Пан] ... [ЗАКРЫТЬ] ...                               │  ← Ряд 5 (нижняя панель)
└───────────────────────────────────────────────────────┘
```

### Справочник кнопок

| Слот | Элемент | ЛКМ | ПКМ | Q (Бросить) |
|---|---|---|---|---|
| 10 | **Включить/Выключить** | Переключить вкл/выкл | — | — |
| 11 | **Тип** (VERTICAL/FIXED) | Переключить тип | — | — |
| 12 | **Выравнивание текста** | Следующее (CENTER→LEFT→RIGHT) | Предыдущее | — |
| 14 | **Прозрачность фона** | +5 | −5 | Сброс до 0 |
| 16 | **Переместить сюда** | Переместить голограмму к вашей позиции | — | — |
| 19 | **Интервал обновления** | +1 сек | −1 сек | Сброс до 0 (LOD) |
| 20 | **Интервал смены страниц** | +1 сек | −1 сек | Сброс до 0 (выкл) |
| 22 | **Дальность видимости** | +5 блоков | −5 блоков | Сброс до дефолта |
| 28 | **Угол направления** *(только FIXED)* | +5° | −5° | Сброс до 0° |
| 29 | **Двусторонняя** *(только FIXED)* | Переключить | — | — |
| 49 | **Закрыть** | Закрыть инвентарь | — | — |

Все изменения сохраняются в YAML-файл голограммы немедленно.

## 13. Локализация (i18n)

Все сообщения плагина и строки GUI хранятся в языковых файлах в папке `plugins/KapyHolograms/`.

### Доступные языки

| Код | Язык | Файл |
|---|---|---|
| `en` | Английский (English) | `messages_en.yml` |
| `ru` | Русский | `messages_ru.yml` |
| `uk` | Украинский (Українська) | `messages_uk.yml` |

### Смена языка

Отредактируйте `config.yml`:
```yaml
language: ru
```

Затем выполните `/kh reload`.

### Настройка сообщений

Откройте соответствующий файл `messages_*.yml`. Все значения поддерживают теги форматирования MiniMessage.

**Структура файла сообщений:**

```yaml
# Префикс перед всеми сообщениями плагина
prefix: "<dark_gray>[<gradient:#ff5555:#ffaa00>KapyHolograms</gradient>]</dark_gray> "

# Сообщения команд (используется синтаксис <плейсхолдер>)
hologram-created: "<green>Голограмма <white><name></white> создана."
hologram-not-found: "<red>Голограмма '<white><name></white>' не найдена."
# ... и т.д.

# GUI-меню — названия предметов и описания
menu-edit-title: "<dark_gray>Голограмма: <gold>{name}"
menu-edit-enabled-name: "<white>Голограмма: {status}"
menu-edit-enabled-on: "<green>ВКЛ"
menu-edit-enabled-off: "<red>ВЫКЛ"
menu-edit-enabled-lore:
  - "<gray>Переключить видимость голограммы"
  - ""
  - "<yellow>ЛКМ <gray>— переключить"
# ... и т.д.
```

### Добавление нового языка

1. Скопируйте `messages_en.yml` в `messages_XX.yml` (замените `XX` на нужный код).
2. Переведите значения.
3. Установите `language: XX` в `config.yml` и выполните `/kh reload`.

## 14. API для разработчиков

KapyHolograms предоставляет чистый Java API для других плагинов, позволяющий создавать голограммы и управлять ими программно.

### Получение API

```java
import dev.kapy.holograms.api.KapyHologramsAPI;
import dev.kapy.holograms.KapyHologramsPlugin;

KapyHologramsAPI api = KapyHologramsPlugin.getApi();
```

### Создание голограммы (Builder)

```java
import dev.kapy.holograms.api.*;

Hologram hologram = api.hologramBuilder()
    .name("myHolo")
    .location(player.getLocation())
    .renderDistance(32.0)
    .line(HologramLine.text("<gold>Привет, мир!"))
    .line(HologramLine.text("<gray>Добро пожаловать, <white>%player_name%"))
    .line(HologramLine.empty())
    .line(HologramLine.item("DIAMOND"))
    .build();

api.getHologramManager().register(hologram);
```

### Добавление страниц программно

```java
HologramPage page2 = hologram.addPage();
page2.addLine(HologramLine.text("<aqua>Страница 2"));
```

### Получение существующих голограмм

```java
HologramManager manager = api.getHologramManager();

// По имени
Hologram h = manager.getHologram("myHolo");

// Все имена
Set<String> names = manager.getRegistry().getNames();
```

### Методы API голограммы

```java
// Показать / скрыть для конкретного игрока
hologram.show(player);
hologram.hide(player);
boolean visible = hologram.isVisibleTo(player);

// Переместить
hologram.setLocation(newLocation);

// Включить/выключить
hologram.setEnabled(false);

// Изменить тип
hologram.setHologramType(HologramType.FIXED);
hologram.setFixedYaw(180f);         // Смотреть на север
hologram.setDoubleSided(true);

// Настройки текста
hologram.setTextAlignment(TextAlignment.LEFT);
hologram.setBackgroundAlpha(128);   // Полупрозрачный фон

// Видимость
hologram.setRenderDistance(64.0);   // Переопределить дефолт

// Интервалы автообновления
hologram.setUpdateIntervalSeconds(5);   // Обновлять содержимое каждые 5 с
hologram.setPageIntervalSeconds(3);     // Автосмена страниц каждые 3 с
```

### Подписка на события

```java
import dev.kapy.holograms.api.event.HologramClickEvent;
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;

public class MyListener implements Listener {

    @EventHandler
    public void onHologramClick(HologramClickEvent event) {
        Player player     = event.getPlayer();
        Hologram holo     = event.getHologram();
        HologramLine line = event.getLine();
        int lineIndex     = event.getLineIndex();  // с нуля

        player.sendMessage("Вы кликнули по строке " + lineIndex + " голограммы " + holo.getName());

        // Отменить стандартные действия по клику
        event.setCancelled(true);
    }
}
```

```java
import dev.kapy.holograms.api.event.HologramCreateEvent;

@EventHandler

    // Получить только что зарегистрированную голограмму
```

## 15. Часто задаваемые вопросы и устранение неполадок

### В: Плагин не загружается / ошибка при старте сервера

**Проверьте:**
1. Сервер работает на **Paper 1.20.5 или новее** (Spigot не поддерживается — сущности `TEXT_DISPLAY` требуют Paper).
2. Используется **Java 21 или новее** (`java -version`).
3. JAR-файл не повреждён — скачайте заново при необходимости.

---

### В: Голограммы не отображаются

1. Убедитесь, что голограмма **включена**: `/kh info <имя>` → проверьте `Enabled: true`.
2. Проверьте **дальность видимости**: используйте `/kh info <имя>` и сравните с вашим расстоянием.
3. Если `fov-culling: true`, убедитесь, что вы **смотрите** на голограмму.
4. Выполните `/kh reload` после редактирования конфига.

---

### В: Плейсхолдеры PlaceholderAPI отображаются как буквальный `%текст%`

1. Убедитесь, что PlaceholderAPI установлен и включён: `/plugins`.
2. Установите нужное расширение PAPI (например, `/papi ecloud download Player` затем `/papi reload`).
3. Убедитесь, что `placeholders.enabled: true` в `config.yml`.
4. Строка должна содержать `%` — только тогда вызывается PlaceholderAPI.

---

### В: Legacy-коды цветов (`&6`, `&#ff5500`) не отображаются

Измените настройку в конфиге:
```yaml
text-format: legacy
```
Затем выполните `/kh reload`. Обратите внимание: существующие голограммы с тегами MiniMessage (например, `<gold>`) будут отображаться как обычный текст после этого изменения.

---

### В: Как сделать голограмму на стене?

Используйте `type: FIXED` и задайте нужный `fixed-yaw`:
```
/kh edit <имя>
```
В GUI установите **Тип** на `FIXED` и настройте **Угол направления** так, чтобы голограмма смотрела вперёд от стены. Включите **Двустороннюю**, если нужно читать с обеих сторон.

---

### В: Как сделать автоматическое переключение страниц?

Через GUI (**Интервал смены страниц**) или напрямую в YAML-файле:
```yaml
page-interval: 5   # переключать страницы каждые 5 секунд
```

---

### В: Можно ли импортировать голограммы из HolographicDisplays / DecentHolograms?

Автоматически — нет. Вы можете вручную создать файлы голограмм в `plugins/KapyHolograms/holograms/`, следуя формату из [§10](#10-формат-файла-голограммы).

---

### В: Как изменить язык для конкретного игрока?

Выбор языка для отдельных игроков не поддерживается. Язык является общесерверным и настраивается через `language` в `config.yml`.

---

### В: Сервер лагает при большом количестве игроков онлайн

Настройте параметры производительности в `config.yml`:
- Включите **FOV-отсечение** (`fov-culling: true`), чтобы пропускать невидимые голограммы.
- Увеличьте `visibility-throttle-ticks` (например, `5`), чтобы распределить проверки видимости.
- Включите **LOD** и увеличьте `far-update-ticks` (например, `100`), чтобы реже обновлять дальние голограммы.
- Уменьшите `default-render-distance`, чтобы меньше голограмм загружалось для каждого игрока.
